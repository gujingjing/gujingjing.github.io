###Context理解
context中文意思为上下文，在Android中，context意为场景，一个context说白了就是一组和用户相互交互的过程，像打电话、发短信、看视频等等。

我们在开发中遇到的context有三种：

  1. Application 的context
  2. Service 的context
  3. Activity 的context

先说下三者之间的联系，首先他们都是继承context的，appliction和service 都是继承contextWrapper,它是contextImp的实现类。
activity 它继承的是contextThemWraper->继承contextWraper

三者之间的作用域不同:

  • application 作用于整个应用程序
  • service 所用于开启的服务中
  • activity 作用在这个开启的界面中

####启动模式

  1. standerd标准启动模式
    • 先进后出
  2. singleTop单一顶部模式
    • 顶部永远保持一个，不可能顶部连续两个一样的activity
  3. singleTask栈唯一模式
    • 在一个任务栈中，有且只有一个activity
    • 启动这个activity,它上面的所有界面都关闭，该activity返回到栈顶，否则新建一个
  4. singleInstance唯一模式
    • 启动这个activity，会重新开启一个新栈，并且栈中有且只有这一个界面

IntentFilter中过滤信息包含 action 、categary、data
匹配的规则

  1. 同时匹配过滤表中的action、categary、data
  2. 每个过滤表中可以有多个action、categary、data 并各自构成不同的类别，一个intent要同时匹配action类别、categary类别、data类别,才能够完全匹配
  3. 一个activity有多组intentFilter,一个intent只要完全匹配一组intentFilter就算匹配成功
